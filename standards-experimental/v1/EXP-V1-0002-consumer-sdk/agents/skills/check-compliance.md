# Skill: Check Compliance

**Purpose**: Verify that a project complies with its adopted APS standards.

## When to Use

Use this skill when:
- Before committing changes that might affect standard compliance
- During code review to verify standards are maintained
- Setting up CI to validate compliance
- Debugging missing or incorrect artifacts

## Prerequisites

- APS CLI available (`aps` command)
- Project has `.aps/manifest.toml` with adopted standards

## Commands

### Basic Compliance Check

```bash
aps check
```

This verifies:
- Required artifacts exist for each adopted standard
- `.aps/index.json` exists and is valid

### Output Examples

**All passing:**
```
Checking compliance...

code-topology@0.1.0:
  ✅ .topology/ exists

✅ .aps/index.json exists

✅ All checks passed
```

**Missing artifacts:**
```
Checking compliance...

code-topology@0.1.0:
  ⚠️  .topology/ missing

✅ .aps/index.json exists

1 warning(s), 0 errors
```

## Fixing Compliance Issues

### Missing Artifacts

Run `aps sync` to regenerate artifacts:

```bash
aps sync
```

### Missing Index

The index is auto-generated by sync:

```bash
aps sync
```

### Outdated Artifacts

Force regeneration with:

```bash
aps sync --force
```

## CI Integration

Add to your CI workflow:

```yaml
# .github/workflows/aps-check.yml
name: APS Compliance

on: [push, pull_request]

jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install APS CLI
        run: cargo install --path lib/aps/crates/aps-cli
      - name: Check compliance
        run: aps check
```

## Understanding the Index

The `.aps/index.json` file contains:

```json
{
  "schema_version": "1.0.0",
  "project": "my-project",
  "standards": [
    {
      "id": "EXP-V1-0001",
      "name": "Code Topology",
      "slug": "code-topology",
      "version": "0.1.0",
      "artifacts": [".topology/"],
      "skills": ["analyze-topology", "query-coupling"]
    }
  ],
  "total_tokens": 2500
}
```

This enables AI agents to:
- Discover available capabilities
- Estimate context token usage
- Find artifact locations

## Best Practices

1. **Run before commits**: Always `aps check` before committing
2. **Add to CI**: Automate compliance checks in your pipeline
3. **Keep index updated**: Run `aps sync` after manifest changes
4. **Commit artifacts**: Consider committing generated artifacts for reproducibility
