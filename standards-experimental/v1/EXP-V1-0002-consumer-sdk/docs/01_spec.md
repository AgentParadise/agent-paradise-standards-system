## 1. Experiment Goals

This experiment provides a consumer-side SDK for adopting APS standards in downstream repositories. It enables projects to:

1. **Declare adopted standards** via a manifest file (`.aps/manifest.toml`)
2. **Generate required artifacts** from adopted standards
3. **Validate compliance** with adopted standards
4. **Enable agent discovery** via auto-generated index (`.aps/index.json`)

### Problem Statement

Currently, APS is author-focused. There is no standard way for a project to:

- Declare which standards it adopts
- Generate required artifacts (e.g., `.topology/` for code-topology)
- Validate compliance with adopted standards
- Help AI agents understand what standards and skills are available

## 2. Schemas

### 2.1 Consumer Manifest (`.aps/manifest.toml`)

The manifest MUST be located at `.aps/manifest.toml` in the consumer repository root.

#### Schema

```toml
schema = "aps.manifest/v1"

[project]
name = "<project-name>"       # REQUIRED: Project identifier
aps_version = "v1"            # REQUIRED: APS major version

# Standards this repo adopts (keyed by slug)
[standards.<slug>]
id = "<STANDARD-ID>"          # REQUIRED: Full standard ID (e.g., "EXP-V1-0001")
version = "<semver>"          # REQUIRED: Pinned version (e.g., "0.1.0")
source = "<source-uri>"       # REQUIRED: Source location
artifacts = ["<path>", ...]   # OPTIONAL: Generated artifact paths

# Substandards to enable (keyed by parent.profile)
[substandards]
"<parent-slug>.<profile>" = { enabled = true }

# Source overrides for development/dogfooding
[sources.overrides]
"<source-uri>" = { type = "<type>", path = "<path>" }
```

#### Source Types

| Type | Description | Fields |
|------|-------------|--------|
| `submodule` | Git submodule in consumer repo | `path` (relative path) |
| `local` | Local filesystem path | `path` (absolute or relative) |
| `release` | GitHub release artifacts | `url` (optional override) |

#### Example

```toml
schema = "aps.manifest/v1"

[project]
name = "my-project"
aps_version = "v1"

[standards.code-topology]
id = "EXP-V1-0001"
version = "0.1.0"
source = "github:AgentParadise/agent-paradise-standards-system"
artifacts = [".topology/"]

[standards.doc-index]
id = "EXP-V1-0002"
version = "0.1.0"
source = "github:AgentParadise/agent-paradise-standards-system"

[substandards]
"code-topology.rust-adapter" = { enabled = true }
"code-topology.3d-force" = { enabled = true }
"code-topology.mermaid" = { enabled = true }

[sources.overrides]
"github:AgentParadise/agent-paradise-standards-system" = { 
  type = "submodule",
  path = "lib/aps"
}
```

### 2.2 Agent Index (`.aps/index.json`)

The index is auto-generated by `aps sync` and SHOULD be committed to enable agent discovery.

#### Schema

```json
{
  "schema_version": "1.0.0",
  "generated_at": "<ISO8601-timestamp>",
  "project": "<project-name>",
  "aps_version": "v1",
  "standards": [
    {
      "id": "<STANDARD-ID>",
      "name": "<human-readable-name>",
      "slug": "<slug>",
      "version": "<semver>",
      "artifacts": ["<path>", ...],
      "skills": ["<skill-name>", ...],
      "tokens": <estimated-context-tokens>
    }
  ],
  "substandards": [
    {
      "id": "<STANDARD-ID>.<PROFILE>",
      "name": "<human-readable-name>",
      "parent": "<parent-slug>"
    }
  ],
  "total_tokens": <sum-of-all-tokens>
}
```

#### Fields

| Field | Type | Description |
|-------|------|-------------|
| `schema_version` | string | Index schema version (currently "1.0.0") |
| `generated_at` | string | ISO8601 timestamp of generation |
| `project` | string | Project name from manifest |
| `aps_version` | string | APS major version |
| `standards` | array | List of adopted standards |
| `substandards` | array | List of enabled substandards |
| `total_tokens` | number | Sum of all standard token estimates |

#### Example

```json
{
  "schema_version": "1.0.0",
  "generated_at": "2025-12-16T00:00:00Z",
  "project": "my-project",
  "aps_version": "v1",
  "standards": [
    {
      "id": "EXP-V1-0001",
      "name": "Code Topology",
      "slug": "code-topology",
      "version": "0.1.0",
      "artifacts": [".topology/"],
      "skills": [
        "analyze-topology",
        "query-coupling",
        "visualize-architecture"
      ],
      "tokens": 2500
    }
  ],
  "substandards": [
    {
      "id": "EXP-V1-0001.LANG01",
      "name": "Rust Adapter",
      "parent": "code-topology"
    }
  ],
  "total_tokens": 2500
}
```

## 3. CLI Commands

### 3.1 `aps init`

Initialize a consumer manifest in the current directory.

```bash
aps init [--name <project-name>]
```

**Behavior:**

1. Create `.aps/` directory if not exists
2. Generate `.aps/manifest.toml` with project info
3. Generate initial `.aps/index.json`

**Output:**

```
‚úÖ Created .aps/manifest.toml
‚úÖ Created .aps/index.json

Next steps:
  aps add code-topology@0.1.0
  aps sync
```

### 3.2 `aps add`

Add a standard to the manifest.

```bash
aps add <standard>[@<version>] [--source <uri>]
```

**Arguments:**

| Argument | Description |
|----------|-------------|
| `standard` | Standard slug (e.g., "code-topology") |
| `version` | Optional version (defaults to latest) |
| `--source` | Optional source URI override |

**Output:**

```
‚úÖ Added code-topology@0.1.0 to .aps/manifest.toml

Run 'aps sync' to generate artifacts.
```

### 3.3 `aps list`

List adopted standards (consumer context).

```bash
aps list
```

**Output:**

```
Adopted Standards:
  code-topology   EXP-V1-0001  0.1.0  ‚Üí .topology/
  doc-index       EXP-V1-0002  0.1.0  ‚Üí .aps/index.json

Enabled Substandards:
  code-topology.rust-adapter
  code-topology.3d-force
  doc-index.adr
```

### 3.4 `aps sync`

Generate/regenerate artifacts from adopted standards.

```bash
aps sync [--force]
```

**Behavior:**

1. Load manifest
2. Resolve sources (apply overrides)
3. For each standard, invoke artifact generation
4. Regenerate `.aps/index.json`

**Output:**

```
Syncing code-topology@0.1.0...
  ‚úÖ Generated .topology/manifest.toml
  ‚úÖ Generated .topology/metrics/
  ‚úÖ Generated .topology/graphs/

Syncing doc-index@0.1.0...
  ‚úÖ Generated .aps/index.json

‚úÖ Sync complete
```

### 3.5 `aps check`

Validate compliance with adopted standards.

```bash
aps check
```

**Output:**

```
Checking compliance...

code-topology@0.1.0:
  ‚úÖ .topology/manifest.toml exists
  ‚úÖ .topology/metrics/ has required files
  ‚öÝÔ∏è  .topology/graphs/coupling-matrix.json: missing 'layout' field

  ‚úÖ .aps/index.json valid
```

### 3.6 `aps update`

Check for available standard updates (info only).

```bash
aps update [--dry-run]
```

**Output:**

```
Checking for updates...

  code-topology  0.1.0 ‚Üí 0.2.0 available
  doc-index      0.1.0 (up to date)

Run 'aps add code-topology@0.2.0' to update.
```

## 5. Source Resolution

When syncing, the SDK resolves standard sources in this order:

1. **Check overrides**: If `sources.overrides` contains the source URI, use the override
2. **Parse source URI**: Extract type and location from URI format
3. **Resolve path**: Convert to absolute filesystem path

### Source URI Format

```
<type>:<org>/<repo>

Examples:
  github:AgentParadise/agent-paradise-standards-system
  local:/path/to/standards
```
